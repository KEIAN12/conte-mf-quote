# MF Setup — 初回OAuth連携スキル

各社員が自分のMFアカウントで認可を行い、トークンをmacOS Keychainに保存する。

このスキルはMCPサーバー `conte-mf-quote` のツールを呼ぶ：
- `mf_me` — 連携状態の確認
- `mf_setup` — OAuth初回セットアップ実行
- `mf_clear_auth` — 連携解除

※ bashでPythonスクリプトを直接叩く必要はない。MCPツール経由で全て完結する。

## いつ起動するか

- ユーザーが明示的に「MF連携したい」「セットアップ」「/mf-setup」等と発言した
- `mf-quote` スキル実行中に `mf_me` が `authenticated: false` を返した
- 連携解除（`mf_clear_auth`）後の再セットアップ

---

## 実行フロー：初回セットアップ

### ステップ1: 既存状態を確認

まず `mf_me` ツールを呼んで、現状を確認：

- `authenticated: true` の場合：
  → 既に連携済み。`office_name` が入っていれば「すでに「〇〇（事業者名）」で連携済みです。再セットアップしますか？」と確認。不要なら終了。
- `authenticated: false` の場合：
  → 新規セットアップに進む。

※ MF Invoice APIには /me エンドポイントがないため、個人ユーザー名・メールは取れない。
   事業者名だけ取得できる（例: 株式会社CONTE）。
   個人識別は MF管理画面の operator_id で自動記録されている。

### ステップ2: ユーザーに準備を促す

```
MoneyForwardクラウド請求書との連携をセットアップします。

準備：
- 代表から共有された「MF Client Secret」を手元に用意してください
- これからブラウザが自動で開きます。**ご自身のMFアカウント**でログインしてください（代表のではなく）

準備ができたら、Client Secretを入力してください。
（セキュリティのため、メモ帳などにコピーした後は必ず削除してください）
```

Client Secretが手元にない場合は、代表に確認するよう案内して中断する。

### ステップ3: MCPツール `mf_setup` を呼ぶ

ユーザーからClient Secretを受け取ったら、MCPツールを呼ぶ：

```
mf_setup(client_secret="<ユーザーが入力した値>")
```

このツールは内部で以下を自動実行する（ブロッキング処理、最大5分）：
1. localhost:8080 でコールバックサーバー起動
2. デフォルトブラウザでMF認可URLを開く
3. ユーザーの認可を待つ
4. 認可コードをトークンに交換
5. Keychain（service=conte-mf-quote）に保存
6. `/me` で連携ユーザーを確認

呼び出し中、ユーザーには以下を伝える：

```
ブラウザが開きました。以下を進めてください：
1. ご自身のMFアカウントでログイン
2. 「claude-agent がアクセスを求めています」→「許可」
3. "✅ 認証成功" ページが出たら、このタブを閉じてCoworkに戻ってください

最大5分待ちます...
```

### ステップ4: 結果を確認

`mf_setup` の戻り値に `office_name`（事業者名）が入っている。ユーザーに伝える：

```
🎉 セットアップ完了しました。
連携事業者: 株式会社CONTE

以後「見積作って」や /mf-quote で、対話しながら見積書が作れます。
誰が作成・編集したかはMF管理画面の履歴（operator_id）で自動記録されます。
```

`office_name` が空の場合でも、トークン自体は保存できているので「セットアップ完了（事業者情報の取得は後でmf_meで再試行）」とだけ伝える。

---

## 連携解除

ユーザーが「MF連携解除したい」と言ったら、MCPツール `mf_clear_auth` を呼ぶ。返り値の `message` をユーザーに伝える。

MF側のアプリ連携も解除したい場合は、[MFアプリポータル](https://id.moneyforward.com)で連携中アプリ一覧から「claude-agent」を連携解除するよう案内する。

## トラブルシューティング

| 症状 | 原因 / 対応 |
|---|---|
| `localhost:8080が既に使われています` | 開発サーバー等が8080を占有している。停止してから再試行 |
| `タイムアウト（300秒）` | 5分以内に認可画面で「許可」を押せなかった。もう一度 mf_setup を呼ぶ |
| `トークン取得失敗 (HTTP 401)` | Client Secret が間違っている。代表に再確認 |
| 認可画面で「このアプリは許可されていません」 | MF Developer Portal のユーザー側で「アプリ連携」権限が付いていない。代表に権限付与を依頼 |
| ブラウザが開かない | `mf_setup` を `open_browser=false` で呼び、戻ってくる `authorize_url` を手動でブラウザにコピペ |

## セキュリティ注意事項

- Client Secretは会話ログに平文で残さない。MCPツールは引数として1回だけ受け取り、以後はKeychainのみに保持
- トークンはmacOS Keychain（service=`conte-mf-quote`）に保存。平文ファイルには書かない
- スコープは `mfc/invoice/data.read mfc/invoice/data.write` のみ（見積・請求書のみ、他は触らない）
- MCPサーバーはユーザーのMac上でネイティブ起動する。sandboxや外部サーバーにはトークンが送られない

## 実装メモ

MCPサーバーは Cowork がプラグイン起動時に自動で起動する（`.mcp.json` 設定による）。
ユーザーは特別な起動操作不要。
