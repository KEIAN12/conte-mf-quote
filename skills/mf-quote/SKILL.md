# MF Quote — 見積書作成スキル

ユーザーと対話しながらMoneyForwardクラウド請求書に見積書の下書きを作成する。

このスキルはMCPサーバー `conte-mf-quote` のツールを呼ぶ。bashは使わない。

## 使うMCPツール

| ツール | 用途 |
|---|---|
| `mf_me` | 連携状態の確認（Step 0） |
| `mf_search_partners` | 取引先検索（Step 2） |
| `mf_list_departments` | 部門取得（Step 2） |
| `mf_search_quotes` | 過去見積検索（Mode 3複製） |
| `mf_get_quote` | 見積の中身取得 |
| `mf_duplicate_quote` | 見積複製 |
| `mf_create_quote` | 新規下書き作成（Step 7） |
| `mf_update_quote` | メタ情報更新 |
| `mf_add_item` / `mf_delete_item` | 品目編集 |
| `mf_download_pdf` | PDF保存（Step 8） |

## 重要原則（絶対に守る）

1. **発行はしない**。必ず下書きのまま。ユーザーが「発行して」と言っても、このスキルでは行わない（MF管理画面から手動で発行する）
2. **税別/税込を必ず明示確認**する。「18万」と言われたら、必ず「税別¥180,000（税込¥198,000）でよろしいですか？」と確認してから進める
3. **担当者名は見積書に書かない**。会社名（株式会社CONTE）だけ。誰が作ったかはMFの履歴（operator_id）で自動記録される
4. **送信しない**。下書き保存までが仕事。メール送付等はMF管理画面で人間が行う

## 実行フロー

### Step 0: 前提確認（未セットアップの検出）

まず `mf_me` ツールを呼ぶ。

- `authenticated: false` または `needs_setup: true` が返ってきた場合：
  → ユーザーに「まずMF連携が必要です。`/mf-setup` を実行してください」と案内して中断。
- `authenticated: true` の場合：
  → 次へ。連携ユーザー名を覚えておく（最終表示時に使用）。

### Step 1: 作り方を選んでもらう

3つのモードを提示：

```
見積書を作成します。どの方法で作りますか？

  1. おまかせ — 件名と金額から、AIが項目を組み立てます
  2. 骨組みだけ — 基本の項目だけAIが書き、金額は一緒に調整します
  3. 複製 — 過去の見積を元にアレンジします

番号で教えてください（1/2/3）。
```

### Step 2: 宛先を聞く（モード1・2・3共通）

```
宛先はどちらですか？（例: 蒲郡市役所、藤城建具店）
```

受け取ったら `mf_search_partners(query="<入力名>")` を呼ぶ。

- 候補0件 → 「MFに登録されていません。先にMF管理画面で取引先登録してから再度試してください」と伝えて中断
- 候補1件 → 自動選択。partner_id を保持
- 候補複数 → 番号付きリストでユーザーに選ばせる

選んだpartner_idで `mf_list_departments(partner_id)` を呼ぶ。

- 部門1件 → 自動選択。department_id を保持
- 部門複数 → 「どの部門宛にしますか？」と聞く
- 部門0件 → api-endpoints.md のdepartment_id規則に従う（MF仕様）

### Step 3: 件名を聞く

```
件名は？（例: まちづくり市民フォーラム 第3回 配信）
```

### Step 4: 金額目安を聞く（任意）

```
金額の目安はありますか？（例: 18万、15万くらい、スキップもOK）
```

**金額が入力されたら、必ず税別/税込を明示確認する**：

```
「18万」ですね。これは税別¥180,000（税込¥198,000）という理解でOKですか？
（もし税込18万なら「税込で」と教えてください）
```

ユーザーの回答で確定。

### Step 5: モード別処理

#### モード1（おまかせ）

件名のキーワード（配信/CM/ブランディング/Web/動画編集/等）を読み取り、`references/quote-templates.md` をReadして該当テンプレートを参照して項目を生成。

金額目安がある場合：税抜目標金額に合わせて各項目の単価を配分。金額目安がない場合：標準的な相場で項目を組み、ユーザーに確認。

#### モード2（骨組みだけ）

最低限必要な項目（件名カテゴリに応じた3〜5個程度）だけ、金額空欄で提示。ユーザーが対話で金額を埋めていく。

#### モード3（複製）

```
複製元を検索します。キーワードを教えてください（宛先・件名どちらでも）：
```

受け取ったら `mf_search_quotes(query="<キーワード>")` を呼ぶ。候補を番号付きで提示（宛先・件名・金額・日付が分かる形）。

選ばれた見積のquote_idで `mf_duplicate_quote(quote_id)` を呼ぶ。返ってきた新しいquote_idを保持。

その後、ユーザーの指示に従って項目や金額を修正：
- メタ情報（件名・日付）変更 → `mf_update_quote`
- 品目追加 → `mf_add_item`
- 品目削除 → `mf_delete_item`

### Step 6: 最終プレビュー

下書き保存する前に、ユーザーに内容を確認させる：

```
以下の内容で下書き保存します：

  宛先: 蒲郡市役所（まちづくり課）
  件名: まちづくり市民フォーラム 第3回 配信
  発行日: 2026-04-20
  有効期限: 2026-05-20（発行日+30日）
  税率: 10%

  項目:
    事前準備・会場下見              ¥18,000 × 1式
    機材費一式（カメラ等）          ¥40,000 × 1式
    機材運搬・設営・撤収費          ¥15,000 × 1式
    配信オペレーター費              ¥8,000 × 4時間 = ¥32,000
    音響費                          ¥15,000 × 1式
    アーカイブ編集・納品費          ¥10,000 × 1式
    進行管理費（打合せ込み）        ¥20,000 × 1式
                                   ──────────
                           小計     ¥150,000
                           消費税   ¥15,000
                           合計     ¥165,000 (税込)

  この内容で下書き保存していいですか？
  [OK] [項目を編集] [金額を変更] [キャンセル]
```

### Step 7: 下書き保存

OKならモードによって分岐：

- **モード1・2（新規作成）**: `mf_create_quote(payload={...})`
  - ペイロード構造は `references/api-endpoints.md` 参照
  - 最低限: `department_id`, `quote_date`, `title`, `excise_type`, `items`
  - 返り値から新しいquote_idを取得
- **モード3（複製済み）**: すでに `mf_duplicate_quote` で作成済みなので、必要に応じて `mf_update_quote` / `mf_add_item` / `mf_delete_item` で編集完了

### Step 8: PDFダウンロード

quote_id を使って `mf_download_pdf` を呼ぶ。save_path を省略すると `~/Downloads/見積書_<quote_id>.pdf` に自動保存される。

わかりやすい名前で保存したい場合は `save_path` を明示：

```
mf_download_pdf(
  quote_id="<id>",
  save_path="/Users/<user>/Downloads/見積書_蒲郡市役所_フォーラム配信_<見積番号>.pdf"
)
```

保存完了したら、ファイルリンクとサマリをユーザーに提示：

```
✅ 下書き保存しました。

- 見積番号: 1234
- 合計: ¥165,000（税込）
- PDFを保存: [見積書_蒲郡市役所_フォーラム配信_1234.pdf](computer:///Users/.../Downloads/...)

MF管理画面で中身を確認して、問題なければ発行（メール送付も含めて）はMF上で行ってください。
```

## 詳細リファレンス

- **`references/api-endpoints.md`** — MF Invoice API v3 のQuote作成・更新のJSONペイロード構造（必須フィールド、品目構造、税区分等）
- **`references/quote-templates.md`** — 件名カテゴリ別の項目テンプレート（配信/CM/ブランディング/Web/動画編集等）

APIペイロードの詳細が必要になったタイミングで `api-endpoints.md` を Read。テンプレートが必要なタイミングで `quote-templates.md` を Read。最初から全部読む必要はない。

## エラーハンドリング

| エラー / 戻り値 | 対応 |
|---|---|
| `mf_me` が `authenticated: false` | `mf-setup` スキルへ誘導 |
| `MFAPIError` HTTP 401 | MCPサーバー側で自動リフレッシュされるはず。それでも失敗なら再セットアップを案内 |
| `MFAPIError` HTTP 422 | エラーメッセージをユーザーに示し、どのフィールドが問題か特定して修正 |
| 取引先0件 | MF側で先に登録するよう案内 |
| 部門が見つからない | 取引先の初期部門（デフォルト）を使うか、MF側で登録するよう案内 |
