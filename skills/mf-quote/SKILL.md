# MF Quote — 見積書作成スキル

ユーザーと対話しながら見積書を組み立てて、MoneyForwardクラウド請求書に下書きとして保存する。

## UX原則（必ず守る）

このスキルを使うのは**開発者ではない社員**も含まれる。以下を徹底する：

1. **専門用語は出さない** — 「API」「エンドポイント」「payload」「quote_id」「MCPツール」等の言葉はユーザーに一切見せない
2. **まずテキストで見積を仕上げる** — MFへの反映は最後。テキスト上で金額・項目を確定してから
3. **反映は必ずユーザーの明示指示で** — 「反映して」「保存して」「MFに入れて」「登録して」など。自動で保存しない
4. **PDFダウンロードは頼まれた時だけ** — 「PDFで」「ダウンロードして」「PDFほしい」等の明示指示がない限り実行しない
5. **発行はしない** — 下書きのみ。発行・メール送付は MF 管理画面で人間がやる
6. **担当者名は見積書に書かない** — 会社名（株式会社CONTE）のみ

## 実行フロー

### Step 0: 連携チェック（裏で静かに）

最初に内部的に `mf_me` を呼んで連携状態を確認する。
- 未連携なら：「まだMFと連携されていません。先に `/mf-setup` で連携してください」とだけ伝えて終了
- 連携済みなら：普通に始める。このチェックをユーザーに見せない

### Step 1: 作り方を聞く

```
見積書を作りますね。どの作り方がいいですか？

  1. おまかせ — 件名と金額を教えてもらえれば、項目は私が考えます
  2. 骨組みだけ — 必要な項目だけ出して、金額は一緒に決めましょう
  3. 複製 — 過去の見積を元に新しく作ります

番号でもキーワードでもOKです。
```

### Step 2: 宛先を聞く（モード1・2・3共通）

```
宛先（お客様）はどちらですか？
```

内部で取引先を検索。候補が複数あれば番号付きで提示して選んでもらう。候補0件なら：

```
MFに登録されていないお客様のようです。先にMF管理画面から取引先を登録してきてください。
```

部門がある取引先の場合は：

```
どの部門宛にしますか？
  1. 担当課
  2. 総務課
```

### Step 3: 件名を聞く

```
件名は？
```

### Step 4: 金額の目安を聞く

```
金額の目安はありますか？（「18万」「15万くらい」みたいに大雑把でOK、分からなければスキップできます）
```

**金額が出てきたら税別/税込を必ず確認**：

```
「18万」ですね。これは税別180,000円（税込198,000円）でいいですか？
もし税込18万だったら教えてください。
```

### Step 5: モード別に項目を組み立てる（テキスト上で）

**ここは MF にはまだ一切触らない。テキストでプレビューを見せるだけ。**

#### モード1（おまかせ）

件名のキーワード（配信／CM／ブランディング／Web／動画編集／撮影／AI／グラフィック等）を判定し、`references/quote-templates.md` をReadして該当テンプレートを参照。金額目安があれば税抜目標に合わせて各項目を配分。

#### モード2（骨組みだけ）

最低限の項目（カテゴリに応じて3〜5個）を金額空欄で出して、ユーザーと対話で金額を埋めていく。

#### モード3（複製）

```
複製したい過去の見積を探します。キーワードを教えてください（宛先・件名・案件名どれでもOK）：
```

検索結果を番号付きで提示して選んでもらう。選ばれたら**内部的に** `mf_get_quote` で中身を取得し、**テキスト上で**「この見積をベースにしますね」と提示。**まだ複製はしない**。

### Step 6: テキスト上でプレビュー & 調整

以下のような形で、MF に入れる前に全体を見せる：

```
こんな感じで組み立ててみました：

  宛先: 〇〇市役所（担当課）
  件名: 市民フォーラム 第3回 配信
  発行日: 2026-04-20
  有効期限: 2026-05-20
  税率: 10%

  項目:
    事前準備・会場下見              ¥18,000 × 1式
    機材費一式（カメラ等）          ¥40,000 × 1式
    機材運搬・設営・撤収費          ¥15,000 × 1式
    配信オペレーター費              ¥8,000 × 4時間 = ¥32,000
    音響費                          ¥15,000 × 1式
    アーカイブ編集・納品費          ¥10,000 × 1式
    進行管理費（打合せ込み）        ¥20,000 × 1式
                                   ──────────
                           小計     ¥150,000
                           消費税   ¥15,000
                           合計     ¥165,000 (税込)

この内容でよろしければ「反映して」と言ってください。
項目を足したい・消したい、金額を変えたい、件名を変えたい等あればそのまま教えてください。
```

ユーザーからの編集指示（「進行管理費を3万に」「音響費いらない」「件名を変えて」等）は、**テキスト上のプレビューを更新するだけ**で対応。MF にはまだ触らない。

### Step 7: MF に反映する（ユーザーが明示指示したら）

ユーザーが以下のような言葉を出した時だけ、MF への保存を実行：
- 「反映して」「反映お願い」
- 「保存して」「保存」
- 「MFに入れて」「MFに登録して」「登録して」
- 「作って」「これで作って」
- 「OK」「OKです」（直前にプレビューを見せて「これでよければ反映してと言ってください」と案内した文脈で）

それ以外の曖昧な返事（「うーん」「いいかも」等）では反映しない。必ずもう一度確認する：

```
MFに反映していいですか？「反映して」と言ってもらえれば進めます。
```

反映処理:
- **モード1・2（新規作成）**: `mf_create_quote` で新規作成
- **モード3（複製）**: `mf_duplicate_quote` で複製→必要に応じて `mf_update_quote` / `mf_add_item` / `mf_delete_item` で編集

### Step 8: 反映完了を報告（URL を必ず提示）

保存が成功したら、返り値の `web_url` を使ってユーザーに報告：

```
✅ MFに下書きで保存しました。

  見積番号: 1234
  合計: ¥165,000（税込）
  MFで開く: https://invoice.moneyforward.com/quotes/1234

中身を確認して、問題なければ MF 管理画面から発行してください。
```

**PDFダウンロードは自動でしない**。ユーザーが「PDFで」「ダウンロードして」等と言った時だけ実行する。

### Step 9: PDFダウンロード（依頼された時だけ）

ユーザーが明示的に依頼したら `mf_download_pdf` を実行して、保存先を教える：

```
📄 PDF保存しました: ~/Downloads/見積書_〇〇市役所_フォーラム配信_1234.pdf
```

## 詳細リファレンス

- **`references/api-endpoints.md`** — MFへの送信内容の構造（必須フィールド、品目構造、税区分）
- **`references/quote-templates.md`** — 件名カテゴリ別の項目テンプレート

必要になったタイミングでReadすればOK。最初から全部読む必要はない。

## エラー時の伝え方

ユーザーには**なぜ失敗したか**だけ分かる言葉で伝える。内部エラーコード（HTTP 401 等）はそのまま見せない。

| 内部で起きたこと | ユーザーへの伝え方 |
|---|---|
| `mf_me` が未認証 | 「まだMFと連携されていません。`/mf-setup` で連携してください」 |
| 401 / 認証エラー | 「MFとの連携が切れているようです。`/mf-setup` で再連携をお願いします」 |
| 422 / バリデーション | 「〇〇が足りない／おかしいようです。〇〇を教えてもらえますか？」 |
| 取引先0件 | 「MFに登録されていないお客様です。先にMF管理画面から取引先登録してください」 |
| ガードエラー（ロック済み等） | 「この見積はすでに発行済み／ロック済みで編集できません」 |

## 補足運用ルール

- 会社の定番は税込18万前後の定額案件（配信、CM、小規模ブランディング）
- 担当者名は見積書本体には入れない。MF管理画面上の「作成者・編集者」で自動追跡される
- 「下書きから発行」は基本的に代表が MF 管理画面でやる（または作成者が確認を取ってから発行）
