# CONTE MF見積作成プラグイン

MoneyForwardクラウド請求書に、AIとの対話で見積書を作成するCoworkプラグイン。

宛先・件名・金額を伝えるだけで、AIが項目を提案し、MFの下書きに保存＋PDFダウンロードまでやる。発行は基本的にMF管理画面から人間が行う（下書き止まり）。

## 誰が使うためのもの

- **吉口慶**（CONTE代表 / KEI）
- **山田知佳**（プロデューサー）
- **大橋友里 / 西村**（デザイナー）
- 今後増える社員

各自が自分のMFアカウントでOAuth連携するため、「誰が作成・編集したか」はMF管理画面の履歴（operator_id）で自動追跡される。見積書本体には担当者名は記載されない（会社名のみ）。

## インストール（各自のMacで1回だけ）

### 1. GitHub Personal Access Token（PAT）をCoworkに登録

代表から配布された GitHub PAT を Cowork の設定画面に登録します（Zoomで画面共有しながら一緒にやります、10分）。

### 2. マーケットプレイスを追加

Cowork の「プラグイン」画面で「マーケットプレイスを追加」を選び、代表から共有された GitHub リポジトリ URL を貼り付けます：

```
https://github.com/KEIAN12/conte-mf-quote
```

### 3. プラグインをインストール

マーケットプレイス一覧に「conte-plugins」が現れるので、その中の「conte-mf-quote」の「インストール」ボタンを押します。

### 4. 初回セットアップ（OAuth連携）

Coworkを開いて、こう入力：

```
/mf-setup
```

対話が始まるので、以下の流れで進める：

1. **Client Secretを入力**（代表から口頭・パスワードマネージャー等で共有されたもの）
2. **ブラウザが自動で開く** → ご自身のMFアカウントでログイン
3. **「claude-agent がアクセスを求めています」** → 「許可」を押す
4. ブラウザに「✅ 認証成功」と出たら、タブを閉じてCoworkに戻る
5. Cowork画面で「🎉 セットアップ完了」メッセージと連携メールアドレスを確認

これで準備完了。以後いつでも見積作成ができる。

## 使い方

### 基本（3つのモード）

Coworkで以下のように話しかける：

```
見積書を作りたい
```

または：

```
/mf-quote
```

すると3つの作り方から選ぶ：

1. **おまかせ** — 件名と金額から、AIが項目を組み立てる
2. **骨組みだけ** — 基本の項目だけAIが書き、金額はユーザーが調整
3. **複製** — 過去の見積を元にアレンジ

### 例：配信案件の新規作成（おまかせモード）

```
ユーザー: 見積作って
AI: どの方法で作りますか？ 1/2/3
ユーザー: 1
AI: 宛先は？
ユーザー: 蒲郡市役所
AI: （MF検索） 候補が見つかりました: 蒲郡市役所 - まちづくり課、蒲郡市役所 - 総務課...
    どの部門ですか？
ユーザー: まちづくり課
AI: 件名は？
ユーザー: 市民フォーラム第3回 配信
AI: 金額の目安はありますか？
ユーザー: 18万ぐらい
AI: これは税別¥180,000（税込¥198,000）でよろしいですか？
ユーザー: OK
AI: （項目を提案）
    - 事前準備・会場下見 ¥20,000
    - 機材費一式 ¥45,000
    - 機材運搬・設営・撤収費 ¥18,000
    - 配信オペレーター費 ¥8,000×4時間
    - 音響費 ¥17,000
    - アーカイブ編集・納品費 ¥12,000
    - 進行管理費 ¥36,000
    小計¥180,000 / 税込¥198,000
    この内容で下書き保存していいですか？
ユーザー: OK
AI: ✅ 下書き保存しました。PDFも保存しました: [見積書_蒲郡市役所_...]
```

```

```

## よくある質問

### Q. 自分の作った見積は他の人からも見える？

A. **はい、社内で共有されます**。MFクラウド請求書は事業者（株式会社CONTE）単位で見積・請求書が共有されるため、同じ事業者の全メンバーから閲覧可能です。ただし、**誰が作成・編集したかは operator_id で記録**されるので、変更履歴は追跡できます。

### Q. 自分が作った見積書に「山田知佳」と印刷される？

A. **いいえ**、見積書本体には会社名「株式会社CONTE」のみ記載されます。担当者名は書かれません。MF管理画面上の「作成者・編集者」の欄には各自の名前が出ます。

### Q. 見積の発行（メール送信）もこのプラグインでできる？

A. **いいえ**、下書き保存までが仕事です。発行・送信は MF管理画面 から行ってください（誤送信防止のため、人間の操作を必須としています）。

### Q. 間違えて下書きを作ってしまった

A. MF管理画面で該当の見積書を削除してください。

### Q. Client Secretが漏れたら？

A. 代表（吉口さん）に連絡。MF Developer Portalで claude-agent アプリの Client Secret を再発行すれば、旧Secretは無効化されます。その後、各自 `/mf-setup` で再セットアップしてください。

### Q. 連携を解除したい

A. Cowork で「MF連携解除して」と話しかけるか、手動で macOS Keychain の「conte-mf-quote」エントリを削除してください。
MF側のアプリ連携も切りたい場合は、[MFアプリポータル](https://id.moneyforward.com) の「連携中アプリ」から claude-agent を連携解除。

## トラブルシューティング

| 症状 | 対応 |
|---|---|
| 「トークンが保存されていません」 | `/mf-setup` を実行して初回セットアップ |
| ポート8080エラー | 開発サーバー等がポート8080を使ってないか確認。停止してから再試行 |
| 認可画面で「このアプリは許可されていません」 | 代表にユーザー権限（アプリ連携）の付与を依頼 |
| 取引先が見つからない | MF管理画面から先に取引先・部門を登録してから再度試す |

## 仕組み（技術者向け）

- アーキテクチャ: **MCPサーバー型プラグイン**
  - Coworkがプラグイン読み込み時に、ユーザーのMac上でMCPサーバー（Python stdio）をネイティブ起動
  - Claudeはスキル内から `mf_setup` / `mf_create_quote` 等のMCPツールを呼ぶ
  - 実装: Python 3 標準ライブラリのみ（外部依存なし）
- OAuth 2.0 Authorization Code flow（CLIENT_SECRET_BASIC）
- トークンは macOS Keychain に保存（`security` CLI 経由、service=`conte-mf-quote`）
- スコープ: `mfc/invoice/data.read mfc/invoice/data.write`（最小権限）
- API: MF Invoice API v3 (`https://invoice.moneyforward.com/api/v3/`)
- ローカルコールバック: `http://localhost:8080/callback`（OAuth初回認可時のみ使用）

### MCPツール一覧

| ツール名 | 用途 |
|---|---|
| `mf_ping` | 疎通確認 |
| `mf_me` | 連携状態・ユーザー情報 |
| `mf_setup` | 初回OAuth連携 |
| `mf_clear_auth` | 連携解除 |
| `mf_search_partners` | 取引先検索 |
| `mf_list_departments` | 部門一覧 |
| `mf_search_quotes` | 既存見積検索 |
| `mf_get_quote` / `mf_duplicate_quote` | 取得・複製 |
| `mf_create_quote` / `mf_update_quote` | 作成・更新 |
| `mf_add_item` / `mf_delete_item` | 品目編集 |
| `mf_download_pdf` | PDF保存 |

## プラグインの更新

代表が GitHub リポジトリを更新すると、Cowork のプラグイン画面で「アップデートを確認」ボタンから最新版に更新できます。Keychainのトークンは維持されるので、再セットアップは不要。
